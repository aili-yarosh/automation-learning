# Завдання для навчання автоматизації тестування

Привіт! Це серія завдань, створених для тебе, щоб ти могла поступово перейти від мануального тестування до автоматизації. Ми почнемо з основ програмування на TypeScript, подібно до того, як вивчають C/C++ у мануалах. Ти створиш репозиторій на GitHub, розмістиш цей файл у README, і будеш створювати папки з проектами для кожного завдання.

## Завдання 0: Створення репозиторія на GitHub

Це завдання — чиста рутина, яка не вимагає придумування логіки. Просто виконай наступні кроки, щоб підготувати середовище для подальших завдань.

1. Створи новий публічний репозиторій з назвою, наприклад, `automation-learning`.
2. Клонуй цей репозиторій на свій комп'ютер за допомогою Git.
3. Скопіюй цей файл README.md у корінь репозиторію.
4. Зроби commit та push на GitHub.

Після цього ти матимеш базу для всіх проектів. Якщо щось незрозуміло, використовуй надані нижче посилання — вони мають повністю задовільнити всі питання.

## Загальні правила
- Для написання коду **ЗАБОРОНЕНО** використовувати AI (ChatGPT, Copilot тощо). Пиши код самостійно, щоб навчитися. Ти можеш гуглити для пошуку інформації та розуміння концепцій, але заборонено копіювати або генерувати код з будь-яких джерел.
- Для організації проекту, створення папок, репозиторія на GitHub та пошуку інформації можна використовувати AI або будь-які ресурси.
- Всі завдання виконуються на TypeScript. Якщо потрібно, встанови Node.js та TypeScript.
- Рішення має бути чистим, зрозумілим і без помилок.
- Надані посилання мають повністю задовільнити всі незрозумілості — використовуй їх як основне джерело знань.

## Чому заборонено використовувати AI на перших кроках?

На цьому етапі навчання важливо розвивати вміння самостійно аналізувати та писати код. Використання AI для генерації коду може завадити цьому процесу, оскільки ти не навчатимешся розбирати алгоритми, розуміти помилки та будувати логіку крок за кроком. Це схоже на вивчення математики: спочатку ти розв'язуєш задачі самостійно, щоб зрозуміти принципи, а потім переходиш до інструментів.

AI буде корисним пізніше, коли ти вже матимеш базу:
- При використанні бібліотек: щоб швидко знайти приклади інтеграції.
- При незрозумілій поведінці коду: для діагностики помилок або пошуку кращих практик.
- Для складних завдань, де логіка відома, але потрібна оптимізація.

Але зараз фокус на самостійному кодуванні — це основа для автоматизації тестування.

## Завдання 1: Сортування масиву пар

### Опис завдання
Створи програму на TypeScript, яка:
1. Читає JSON-файл з масивом об'єктів. Кожен об'єкт має два поля: `number` (число) та `name` (рядок, назва).
2. Сортує цей масив за зростанням за полем `number` (якщо числа рівні, порядок неважливий).
3. Записує відсортований масив у новий JSON-файл.

Використовуй простий алгоритм сортування, такий як bubble sort (міхурове сортування). Це допоможе тобі зрозуміти основи циклів та умов.

### Приклад вхідного файлу (input.json)
```json
[
  {"number": 5, "name": "apple"},
  {"number": 2, "name": "banana"},
  {"number": 8, "name": "cherry"}
]
```

### Приклад вихідного файлу (output.json)
```json
[
  {"number": 2, "name": "banana"},
  {"number": 5, "name": "apple"},
  {"number": 8, "name": "cherry"}
]
```

### Кроки для виконання
1. Створи нову папку для проекту, наприклад `task1-sorting`.
2. Ініціалізуй TypeScript-проект: `npm init -y`, потім встанови TypeScript: `npm install typescript --save-dev`, та ініціалізуй: `npx tsc --init`.
3. Створи файл `input.json` з прикладом даних.
4. Напиши TypeScript-код у файлі `sort.ts`. Кожний крок супроводжується поясненням, як використовувати наявні мануали для його виконання:
   - Імпортуй необхідні модулі (якщо потрібно): Використовуй мануал "Читання та запис файлів у Node.js" для імпорту `fs` модуля за допомогою `import fs from 'fs';`.
   - Напиши функцію для читання JSON: З мануала "Читання та запис файлів у Node.js" візьми приклад читання файлу синхронно (`fs.readFileSync`), потім розпарси JSON за допомогою `JSON.parse`. Визнач типи для об'єктів за мануалом "Робота з JSON у TypeScript".
   - Напиши функцію bubble sort для сортування масиву об'єктів за `number`: З мануала "Алгоритм bubble sort" візьми псевдокод та адаптуй його до масиву об'єктів, порівнюючи `obj.number`. Напиши цикли самостійно, розуміючи логіку.
   - Напиши функцію для запису результату в новий JSON-файл: З мануала "Читання та запис файлів у Node.js" візьми приклад запису файлу (`fs.writeFileSync`), попередньо серіалізувавши масив у JSON за допомогою `JSON.stringify`.
   - Викликай ці функції в основній частині коду: Об'єднай все в послідовність викликів, використовуючи мануал "Початок роботи з TypeScript" для розуміння структури файлу.
5. Скомпілюй та запусти: `npx tsc sort.ts && node sort.js`.
6. Перевір, що `output.json` створений і містить правильно відсортований масив.
